# -RK3588-AI-
针对驿站仓储管理人工成本高、效率低、安全管控弱的行业痛点，本作品基于RK3588主控的边缘AI平台，打造智能仓储驿站管理助手。系统融合机器视觉识别、边缘计算，搭载巡逻模块，精准完成快递码识别、仓储空位检测与货物信息智能录入；配套可视化交互界面，实现数据查询、状态监控一体化操作。同时集成消防隐患检测功能，实时监测环境安全参数，险情触发即刻告警，全方位保障仓储运营安全，有效降低人力成本、提升管理效率。

本作品基于瑞芯微开源的方法，及广大单片机开源资料，及开源的PYQT5库开发而来

PYQT27.py的介绍：
本程序开头大量使用 import 导入各类库文件，是整个仓储巡检系统运行的基础，所有功能都依赖这些模块实现。
首先导入内置基础库：sys 用于程序全局参数管理、程序退出与进程调度；time 用于线程延时、帧率计算与串口时序控制，保障后台任务稳定运行。
其次导入硬件相关库：serial 是串口通信库，负责上位机与下位机 STM32 通信，接收火灾报警、四路电机状态数据，同时下发巡逻启停、小车运动指令。
核心 UI 部分全部依赖 PyQt5 模块：分别导入窗口、布局、按钮、表格、弹窗等界面控件，同时引入多线程、自定义信号、定时器、键盘事件与图像渲染类。此举可以实现全屏触控界面、无卡顿后台线程、弹窗警告、自定义虚拟键盘等交互功能，避免 UI 卡死。
视觉 AI 模块：cv2 用于读取摄像头视频流、画面预处理；YOLOv8 优化推理函数，完成 RKNN 硬件加速目标检测，实现仓储实时视觉巡逻识别。
整套系统基于 PyQt5 搭建全屏触控式仓储管理界面，采用模块化多窗口设计，包含首页主菜单、快递查询、快递录入、仓储巡逻四大独立页面，配套多款自定义弹窗与虚拟键盘组件，适配嵌入式 RK 设备大屏全屏操作，支持 F11 一键切换全屏、ESC 退出全屏 / 关闭窗口。
首页主界面
页面顶部为状态监控栏，实时展示巡逻运行状态、环境安全等级，搭配红绿黄三色状态指示灯；设置功能按钮可手动切换安全等级、模拟设备故障、查看四路电机实时状态。中间白色卡片为功能导航区，三个主按钮分别跳转查询、录入、巡逻页面。串口检测到火灾报警信号时，自动弹出黄色渐变风格的环境异常无边框弹窗，提示工作人员处理险情。
快递查询界面
上半区是数据表格，两列表格展示快递位置码与对应仓储格口，支持模糊码检索、一键刷新全部数据，底部统计当前快递条目总数；下半区内置数字字母虚拟键盘，无需外接实体键盘即可输入查询编码，查询无匹配数据会弹出提示框，同时提供返回首页、一键退出程序按钮，表格行交替底色，表头自适应拉伸，界面整洁易读。
快递录入界面
简约卡片布局，设置位置码、存放位置两个输入框，虚拟键盘仅支持位置码快速录入，存放位置支持外接键盘输入中文。录入时自动校验编码重复，重复编码会弹出警告，录入成功弹窗提示并清空输入框，配备保存、返回首页按钮，整体布局紧凑，适配全屏触控操作。
巡逻控制界面
展示小车实时巡逻状态与状态指示灯，提供开始巡逻、停止巡逻、返回首页三个功能按钮。点击开始巡逻会启动串口下发小车运动指令，同时开启 RKNN-YOLO 视觉检测线程，弹出黑色无边框全屏检测窗口，实时显示摄像头推理画面与实时 FPS 数值，窗口顶部带关闭按钮；停止巡逻仅下发小车停机指令，返回首页会自动关闭视觉线程、释放摄像头与 RKNN 资源。
自定义弹窗组件
包含三类无边框半透明弹窗：红色故障警告弹窗、黄色环境异常弹窗、蓝色电机状态弹窗，统一圆角卡片 + 渐变按钮样式，弹窗自动居中于父窗口，点击确认即可关闭，视觉统一美观。
通用虚拟键盘组件
全局复用数字字母触控键盘，适配查询、录入双页面，支持数字、横杠、字母 A/B 输入，具备退格、清空、确认功能，按键缩小适配大屏，渐变按键区分功能键，专门适配无物理键盘的嵌入式设备场景。
界面底层适配特性
所有页面封装基础窗口父类，统一全局微软雅黑字体，全套渐变背景、阴影卡片美化样式；封装全屏适配工具函数，打开页面自动铺满屏幕；所有硬件读写、AI 推理、串口监听全部放入独立 QThread 子线程，UI 操作不会出现卡顿、画面冻结问题。
                              

stream.py作为项目主运行脚本，加载.rknn推理模型，联动func、pool工具模块完成RKNN环境初始化与NPU多线程调度。程序读取图像后执行目标检测，完成ROI区域空位判定，同步调用绘图接口标注识别结果，最终对接GUI界面实时刷新画面，实现硬件加速推理、多任务并发与可视化展示一体化流程。

下位机 STM32 通过外设采集电路完成 MQ-2 烟雾传感器模拟量、DS18B20 温度传感器单总线数字量的实时读取，实现现场消防环境数据持续监测；内置消防阈值判定逻辑，可根据预设温、烟浓度阈值自主判断消防风险等级。设备依托 TTL 串口 CMD 指令通信协议与 RK3588 上位机建立双向数据交互链路：一方面向上位机实时回传温度、烟雾监测数据与消防预警状态，另一方面接收上位机下发的运动控制指令。程序主函数采用轮询调度架构，循环完成串口指令解析、消防监测数据采集与风险判断、电机驱动控制函数调用三大核心任务，统筹协调车体运动执行与消防安全实时巡检两大功能模块同步运行。
